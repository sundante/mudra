Chapter 01 — Design Philosophy

The Five Principles

🔭
Clarity over decoration
Every visual element must earn its place by making financial data clearer. No decorative charts, no gratuitous gradients. If a user cannot read the key number in under 2 seconds, the design has failed.
⚖️
Gold is sacred
Gold appears only on the most important elements: the primary CTA, the brand mark, the runway number, and the active state. Overuse kills its signal power. Every gold element must justify its presence.
📐
White breathes
Pure white (#FFFFFF) as the base creates maximum contrast for data. Generous whitespace is not emptiness — it is the cognitive space users need to process financial information without anxiety.
📊
Data has a grammar
Green = positive/income. Red = expense/negative/debt. Amber = investment/neutral. Gold = brand/CTA. This colour grammar never breaks — not even in charts. Users learn it once and read it everywhere.
🔢
Numbers are monospaced
Every currency amount — without exception — uses IBM Plex Mono. Monospaced digits align vertically in lists, making comparison instant. This is a hard rule, not a guideline.
🏛️
Hero = Cormorant
The large financial numbers (Runway, Net Worth, Total Assets) use Cormorant Garamond — a serif that feels authoritative and premium at display sizes. It creates the "weight" that big numbers deserve.

Chapter 04 — Data Visualisation

8 Chart Patterns

Every chart serves the clarity principle: one insight, immediately readable. Complex multi-axis charts are banned. Each pattern is defined with its use case and implementation approach.

Chart Type 1 — Semi-Donut (Half Ring)
Use for: asset allocation, budget breakdown, category split. The open bottom provides space for the total amount label.

Flutter: CustomPainter with drawArc() · Gap at bottom = 60° · Each segment = strokeWidth 12–14

Red:            #A83226 — expenses, debts, losses
Red light:      #FDECEA
Amber:          #9A5510 — investments, neutral/pending
Amber light:    #FEF0E0

═══════════════════════════════════════════════
TYPOGRAPHY — USE EXACTLY THESE
═══════════════════════════════════════════════
Display/Hero:   Cormorant Garamond 700 — for large financial numbers
                and screen headings. Elegant serif.
Body/UI:        IBM Plex Sans 400/500/600 — all UI text, labels, body
Amounts/Mono:   IBM Plex Mono 500/600 — EVERY currency amount, no exceptions
                Also: section labels (9.5px, ALL CAPS, tracked 1.8)

═══════════════════════════════════════════════
COLOUR GRAMMAR — NEVER BREAK THESE
═══════════════════════════════════════════════
Positive amounts / income   → #1E6B44 green
Expenses / debts / losses  → #A83226 red
Investments / neutral      → #9A5510 amber
Brand / CTA / active       → #8A6520 gold
Background everywhere      → #FFFFFF white

═══════════════════════════════════════════════
CHART COMPONENTS — USE THESE PATTERNS
═══════════════════════════════════════════════
1. SEMI-DONUT (half ring, bottom open) — asset allocation
2. FUEL GAUGE RING (240° arc, animated) — monthly runway
3. BAR CHART (rounded top, 6-month view) — spend trends
4. INCOME FLOW TREE (horizontal bars by source) — income
5. STACKED BAR (single bar, multi-segment) — budget split
6. SPARKLINE (area fill, no axes) — net worth trend
7. FULL DONUT (hole 55%) — category breakdown
8. PROGRESS ROWS (linear bars) — budget vs actual

═══════════════════════════════════════════════
CARD PATTERNS
═══════════════════════════════════════════════
HERO: Gold gradient dark card (#2A1C04→#A07020). Max 1 per screen.
      Shows: Runway / Net Worth / primary KPI. White text.
DATA: White card, 1px #EBEBEB border, 14px radius, shadow-sm.
      Shows: metrics, charts, lists.
TILE: 2×2 or 3×1 grid of small metric tiles. IBM Plex Mono values.
LIST: White, no outer border, internal dividers only.
      Row: logo circle + name/sub + amount (mono) + % badge.

═══════════════════════════════════════════════
SCREEN STRUCTURE
═══════════════════════════════════════════════
Status bar: white
App bar: white bg, "Mudra" in Cormorant Garamond gold left,
         icon buttons right (notification bell, settings gear)
Body: white scroll area, 16px horizontal padding
      Section labels: IBM Plex Mono 9.5px ALL CAPS ink4 above each section
      Hero card: always first, full width
      Stats tiles: 3-column grid below hero
      Charts: full-width data cards, 10px content padding
      Lists: white cards with 1px borders
FAB: gold circle, + icon, bottom-right, gold drop-shadow
Bottom nav: white, 1px top border, gold active icon + label
Tabs: 5 — Home / Funds / Debits / Portfolio / Settings

═══════════════════════════════════════════════
WHAT NOT TO DO
═══════════════════════════════════════════════
✗ No purple, blue, or teal gradients
✗ No dark mode (light theme only)
✗ No Inter, Roboto, or system fonts
✗ No pie charts (use semi-donut or full donut only)
✗ No currency amounts in sans-serif (always IBM Plex Mono)
✗ No cream/warm background (#FAF8F4) — use pure white
✗ No more than 1 gold gradient card per screen
✗ No "buy", "invest", "apply" CTAs — awareness only
